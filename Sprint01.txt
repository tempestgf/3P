SPRINT-01. Travel Planner Scaffolding
Delivery: 01/03/2025 23.55
Abstract
The first milestone in Mobile Application Development is completing the first sprint
(Sprint I) by building the initial structure of the Travel Planner app.
During these initial weeks, students will create the skeleton of their project with core
elements and essential features.
Objectives
1. Set up an Android Studio project for the Travel Planner.
2. Create and initialize a Git and GitHub repository.
3. Design the core screens of the travel management system.
4. Document the project using Markdown.
5. Implement an initial customization of the app.
6. Create the first release in GitHub. V1.0.0
1. The version must follow Semantic Versioning and be lower than 1.0.0
(development phase) of sprint01.
2. The final submission of Sprint 01 must be published as a GitHub Release
(e.g., v1.0.0).
3. If the project is not published as a Release, it will not be evaluated.
Only the latest published Release will be considered for grading.
Tasks
T1. Project Setup (1 Points)
• T1.1 Create a Product Name.
Students must define an original and creative product name for the application.
The name must:
• Be unique and not already used by existing commercial applications.
• Reflect the purpose or identity of the project.
• Demonstrate originality and intentional design thinking.

Delivery
Students must submit a GitHub link containing a release with all tasks completed. The
release version must follow semantic versioning with MAJOR with the SPRINT number,
(e.g., v1.0.0, v1.2.1). If multiple releases exist, the highest version number will be
graded.
Sprint Documents
Each sprint requires two documents. A template for both is available at:
github.com/vitorlui/DAM_105025_sprints_structure
• plan_sprint01.md — Created at the start of the sprint. It serves as the product
backlog and planning document, outlining tasks and goals, including
assignments for each team member so that everyone contributes.
• final_sprint01.md — Completed at the end of the sprint. It serves as a
retrospective, reviewing what was accomplished, what went well, and what can
be improved.
